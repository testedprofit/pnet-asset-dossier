# PNET Moonshot Runway

Status: public-safe growth and legitimacy plan  
Date: 2026-07-11  
Tone: ambitious, evidence-first, not investment-oriented

## Purpose

This document defines what a "moonshot" should mean for ProfitNet (`PNET`) without implying price targets, guaranteed listings, yield, investment return, or bridge support.

The clean thesis:

```text
PNET can become a visible AI-first Web3 operations experiment by combining public evidence, token metadata, verification tooling, and cautious multichain practice.
```

The project should moonshot the process, not promise the chart.

## The Wedge

The strongest public idea is not:

```text
PNET is another token.
```

The stronger idea is:

```text
PNET is an AI-first token operations lab that shows how a project can launch, verify, document, and test multichain workflows in public.
```

That gives the project a reason to exist beyond the pool.

## Public Positioning

Use:

```text
ProfitNet (PNET) is an AI-first Web3 project exploring public token verification, metadata, community tooling, and cautious multichain interoperability. It remains rooted on Algorand while testing a verified Base ERC-20 expansion.
```

Avoid:

- investment asset,
- passive income,
- hashrate-backed token,
- guaranteed liquidity,
- guaranteed listing,
- live Wormhole bridge,
- Coinbase/Base endorsement,
- burned supply unless supported by public evidence,
- deep liquidity unless the pool actually has depth.

## Runway

### Phase 1: Finish the Public Legitimacy Layer

Goal: make every public surface agree.

Actions:

- Wait for BaseScan token profile/logo review.
- Do not submit duplicate BaseScan applications.
- Update the official website so it references both the Algorand ASA and Base ERC-20.
- Keep the GitHub dossier as the source of truth.
- Publish a simple public status page with explorer, pool, logo, and dossier links.
- Add a token-list JSON only after the logo/profile details are stable.

Proof targets:

- BaseScan profile/logo approved,
- official website updated,
- no conflicting token descriptions,
- public links updated in this dossier.

### Phase 2: Build the ProfitNet Command Center

Goal: turn PNET from a token page into a useful tool surface.

Minimum first version:

- Base contract status,
- Algorand ASA status,
- Uniswap pool link,
- BaseScan profile status,
- latest dossier commit,
- logo/media links,
- public claims checklist,
- Wormhole NTT lab status,
- clear non-investment disclaimer.

No wallet connection is required for the first version. The first product should help a stranger understand the project safely.

### Phase 3: Ship the AI Verifier

Goal: make the AI-first claim real.

Prototype:

- user enters a Base ERC-20 contract or Algorand ASA ID,
- tool checks public facts and risk labels,
- tool generates a Markdown dossier,
- tool flags missing evidence,
- tool links directly to explorers and public sources,
- PNET is used as the reference case study.

This moves the project from "token with docs" to "docs as a product."

### Phase 4: Distribution Without Spam

Goal: become discoverable through useful public work.

Channels:

- GitHub releases and changelog posts,
- X/Twitter build logs,
- Farcaster/Base community updates,
- Algorand community updates,
- short educational threads about token verification,
- public demo videos for the Command Center and AI verifier.

Message pattern:

```text
We are building PNET as an AI-first Web3 verification and multichain-ops experiment. Here is the verified Base contract, Algorand root, public dossier, seed pool, and the next NTT testnet lab.
```

Do not ask communities to spam listing pages or review queues.

### Phase 5: Liquidity and Market Readiness

Goal: improve trading access without pretending the market is mature.

Actions:

- Keep one canonical Uniswap pool reference.
- Add liquidity only according to a written plan.
- Track slippage, TVL, swaps, volume, and holders as dated snapshots.
- Consider Base-native liquidity venues only after the public profile, site, and docs are stable.
- Avoid wash trading or artificial activity.
- Keep price/volume claims dated and sourced.

Listing gates:

- public website updated,
- token profile/logo approved,
- active public pool,
- non-trivial organic trading activity,
- official contact surface,
- consistent supply and contract records.

### Phase 6: Wormhole NTT Practice

Goal: prove bridge mechanics before making bridge claims.

Actions:

- Run testnet NTT first.
- Use tiny testnet transfers only.
- Record manager addresses, modes, rate limits, peers, and tx hashes.
- Keep Base as the canonical ERC-20 surface unless a formal migration plan says otherwise.
- If Algorand is used, verify the current supported NTT path and record exact chain names and tooling.

Approved public wording:

```text
PNET is preparing Wormhole NTT practice workflows. No production PNET bridge is live yet.
```

### Phase 7: Official Listing Pipeline

Goal: use evidence, not middlemen.

Free/official surfaces:

- BaseScan token profile/logo update,
- Blockscout and Sourcify verification,
- Uniswap public pool,
- public token-list JSON,
- CoinGecko request only after active-market prerequisites are credible,
- CoinMarketCap request only when volume/activity and project evidence are no longer trivial.

Do not pay unofficial listing brokers. Use official forms and keep copies of the submitted evidence.

## Anti-Claims

PNET should not claim:

- guaranteed returns,
- passive income,
- exchange listing certainty,
- bridge support before proof,
- Base/Coinbase endorsement,
- Algorand Foundation endorsement,
- hashrate rewards,
- deep liquidity,
- verified burned supply without transaction evidence,
- or any right to revenue, treasury, compute, hashrate, or future value.
