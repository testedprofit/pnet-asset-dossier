# ProfitNet Content and Marketing System

Status date: 2026-06-27

This folder contains public-safe content systems for ProfitNet / PNET.

The goal is education, verification, and product literacy. Content should teach people how to inspect asset facts, use public tools, understand tested workflows, and participate in the community without hype.

## Positioning

ProfitNet should communicate as:

- verification-first,
- practical,
- source-linked,
- calm,
- useful to builders,
- useful to reviewers,
- honest about unknowns.

ProfitNet should not communicate as:

- a guaranteed-return opportunity,
- passive income,
- a managed strategy,
- copy trading,
- an exchange-listing path,
- a price prediction,
- a deposit product,
- a shortcut around independent review.

## Content Pillars

| Pillar | Purpose | Example |
| --- | --- | --- |
| Verification education | Teach people how to check claims | How to verify any Algorand ASA tokenomics |
| Tool tutorials | Teach repeatable workflows | How to read a Vestige asset page |
| Tested methods | Document repeatable public workflows | A 10-minute ASA verification routine |
| Transparency updates | Show what changed and what remains open | Monthly dossier update |
| Community contribution | Explain access/reputation credits safely | What app-local contribution credits are |
| Builder resources | Help developers use public data | Read-only indexer review checklist |

`Tested methods` means repeatable verification and workflow methods. It does not mean guaranteed trading outcomes, guaranteed earnings, or investment performance.

```markdown
Educational content only. No investment advice, no deposit request, no passive-income claim, no guaranteed value, and no exchange-listing claim. Verify public sources before relying on any asset, wallet, market, or tokenomics statement.
```
