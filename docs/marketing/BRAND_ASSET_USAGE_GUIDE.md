# ProfitNet Brand and Asset Usage Guide

Status date: 2026-06-27

This guide keeps ProfitNet brand usage aligned with the transparency-first culture.

## Brand Foundation

| Element | Guidance |
| --- | --- |
| Brand names | `testedprofit`, `ProfitNet`, `Profitnet` where context requires |
| Token name | `ProfitNet` |
| Unit | `PNET` |
| Tone | Clear, calm, technical, practical |
| Primary promise | Better verification, better documentation, better workflows |
| Avoid | Hype, price language, guaranteed outcomes, passive-income framing |

## Core Message

Use:

```text
ProfitNet helps people verify public asset facts, tokenomics claims, and market-intelligence records with source-linked documentation.
```

Avoid:

```text
ProfitNet helps users earn, profit, or receive guaranteed returns.
```

## Tagline Options

Approved:

- Verify first. Build better.
- Public facts. Clear methods.
- Source it before you trust it.
- Market intelligence, documented.
- Better records for better review.

Avoid:

- Guaranteed profit.
- Earn with PNET.
- Passive income for holders.
- The next listing.
- Risk-free crypto intelligence.

## Logo Usage

Use the ProfitNet / PNET logo only in contexts that are:

- official project pages,
- documentation,
- educational content,
- tutorial thumbnails,
- verification reports,
- community updates.

Do not use the logo:

- beside price predictions,
- beside guaranteed-return claims,
- beside exchange-listing speculation,
- beside deposit prompts,
- inside wallet screenshots,
- inside third-party claims that are not verified.

## Visual Style

| Element | Guidance |
| --- | --- |
| Layout | Clean, readable, source-forward |
| Screenshots | Use public pages only; redact private data |
| Charts | Include source and date |
| Tables | Prefer tables for claims, status, and evidence |
| Badges | Use `Verified`, `Snapshot only`, `Needs verification`, `Not claimed` |
| Colors | Use brand colors consistently; avoid alarmist red/green market framing |

## Jingle / Sonic Brand

The sonic brand should feel like a short verification cue, not a trading prompt.

Approved 3-second audio direction:

```text
Soft digital chime, two clear notes, then a subtle resolved tone.
Mood: precise, calm, reliable.
```

Approved spoken bumper:

```text
ProfitNet. Verify first.
```

Approved longer bumper:

```text
ProfitNet. Public facts. Clear methods.
```

Avoid spoken lines:

- Earn with ProfitNet.
- Profit starts here.
- Guaranteed gains.
- Passive income made simple.
- Buy before it lists.

## Video Intro Script

```text
Today we are reviewing [topic] using public sources.

We will record the source, capture date, observed value, method, and limitation.

This is educational content only. Verify public sources before relying on any asset, wallet, market, or tokenomics claim.
```

## Thumbnail Text

Approved:

- Verify This ASA
- Tokenomics Checklist
- ASA Controls Explained
- Market Snapshot Review
- Needs Verification
- Public Source Walkthrough

Avoid:

- Huge upside
- Passive income
- Guaranteed return
- Listing soon
- Risk free
- Deposit and earn

## Brand Review Checklist

- [ ] Does the asset use official logo files only?
- [ ] Does the copy avoid investment and listing claims?
- [ ] Are source links included where facts are stated?
- [ ] Are market metrics dated?
- [ ] Are unknowns marked?
- [ ] Are screenshots free of private wallet data?
- [ ] Does the content teach a repeatable method?
