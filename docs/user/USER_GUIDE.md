# PNET User Guide

Status date: 2026-06-27

This guide explains how a public reader should use the PNET dossier.

PNET documentation is designed for verification, not persuasion. A claim should be treated as unresolved unless the dossier provides a source link, date, method, and status label.

## What PNET Is

PNET is the unit name for the ProfitNet Algorand Standard Asset with ASA ID `3169177585`.

The dossier is a public review package for:

- asset facts,
- tokenomics claims,
- market snapshots,
- public links,
- verification steps,
- media records,
- public claims policy,
- future contribution protocol documentation.

## What PNET Is Not

PNET documentation does not provide:

- investment advice,
- trading advice,
- guaranteed return claims,
- passive-income claims,
- exchange-listing guarantees,
- bridge guarantees,
- custody instructions,
- deposit instructions.

## How To Read Status Labels

| Label | Meaning |
| --- | --- |
| Verified | Evidence, date, method, and observed value are recorded |
| Snapshot only | Observed on a date, but may no longer be current |
| Needs verification | Recorded but not independently confirmed |
| Needs on-chain verification | Must be checked against chain, explorer, indexer, account, app, or pool data |
| Needs methodology review | A value exists, but the calculation method is unclear |
| Not claimed | The dossier intentionally makes no claim |

## Safe Participation Model

The planned PNET Community Contribution Protocol is framed as a non-custodial application for app-local contribution credits.

Contribution credits:

- may support access or reputation features,
- are not PNET tokens,
- are not deposits,
- are not yield,
- are not passive income,
- are not guaranteed value,
- are not claims on treasury assets.

Any public deployment should publish the app ID, source, ABI, deployment transaction, admin policy, receipt policy, tests, and audit status before being described as live.

## What To Check Before Trusting A Claim

| Question | Good answer |
| --- | --- |
| Is there a source link? | Public explorer, indexer, project page, DEX page, or repository file |
| Is there a date? | Capture date is visible |
| Can the check be repeated? | Method is described |
| Is it stable or stale? | Market data is marked as snapshot only |
| Is the claim about value or income? | The dossier should avoid or reject it |
| Does the claim involve a wallet label? | It should require human confirmation |
| Does the claim involve a contract? | App ID, source, ABI, tx, and audit status should be present |
